# AirMosaic AI Web Design System Proposal

This proposal must be confirmed before visual page implementation starts.

## Recommended Direction

AirMosaic AI should feel like a serious research and decision platform rather than a marketing website.

## Proposed Design Decisions

- Color palette:
  - Primary: deep atmospheric blue-green for trust and environmental context.
  - Secondary: clean neutral grays for data-heavy surfaces.
  - Accent: measured amber or cyan for alerts, active layers, and model outputs.
- Typography:
  - Heading: a precise sans-serif with a technical but not generic feel.
  - Body: readable sans-serif optimized for dense dashboards.
  - Code/data: monospace for API examples and model traces.
- Spacing system:
  - Compact dashboard spacing based on 4px and 8px increments.
  - Larger spacing only for the first-screen overview.
- Border radius:
  - Mostly 6px to 8px for cards and panels.
  - Smaller radius for dense controls.
- Shadow hierarchy:
  - Minimal shadows.
  - Prefer borders, muted fills, and clear layout over floating-card decoration.
- Motion style:
  - Subtle state transitions under 180ms.
  - No decorative animation unless it explains workflow or model progress.

## First Screen Concept

The first screen should show the actual platform surface:

- Left: external Agent task input and tool-call examples.
- Center: data catalog and geospatial/indicator preview.
- Right: model design and causal design trace.

No pure landing-page hero should be used unless a separate public marketing site is requested.
